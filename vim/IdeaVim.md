下一个重要的文献是 “黑三角”。它应该是一个相对简短的指南，帮助你启动并运行你的项目，方便用户自由探索。

你的“黑三角”文档应是简短指南，引导用户了解检索、安装和戳入你的项目或语言的过程。

API 文档就像汽车的用户手册。当轮胎出了问题需要更换时，这简直是天赐良机。但如果你正在学开车，这对你没什么帮助，因为人们不会通过阅读按字母顺序排列的无关信息列表来学习 。

这让我想到下一种类型的文档：“毛球”。现在用户应该已经看过“首次接触”文档和“黑三角”文档了。你已经让他们上瘾并准备学习，但他们仍是新手。
“毛球”是那个扭曲、纠结的教学迷宫，将把这些新手变成专家级使用者。它会一点点引导他们的思维，直到他们对你的项目运作有相当深入的理解。
